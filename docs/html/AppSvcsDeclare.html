Function: wrapperAjv¶

prepare an instance of ajv to validate request payloads. Upon success leaves ajv instance in this.requestValidator. Throws on error. this.requestValidator is synchronous; the request schema ignores declaration contents (which we run through as3parser later)

wrapperAjv()¶

Function: declarationIndex¶

list all locally-saved declarations (if any) (relies on caller to set ‘this’)

declarationIndex()¶
Return array:
  • possibly empty

Function: savedDeclaration¶

return a Promise to obtain the saved declaration (if any) for target + age. If age is zero and no local copy is available and target seems reachable, try to get stored declaration from target device.

savedDeclaration(controls, target[, age])¶
Arguments:
  • controls (object) – info needed to access BIG-IP
  • target (string) – (please normalize loopback-IP to “localhost”)
  • age (number) – zero is most recent
Return Promise:
  • resolves to declaration or undefined

Function: tenantFilter¶

exclude non-selected Tenants from a declaration. SIDE EFFECT: modifies decl

tenantFilter(decl, tenants, restOperation)¶
Arguments:
  • decl (object) – exclude non-selected Tenants from a declaration. SIDE EFFECT: modifies decl
  • tenants (array) – empty is okay
  • restOperation (object) – exclude non-selected Tenants from a declaration. SIDE EFFECT: modifies decl
Return boolean:
  • true upon success

Function: onStart¶

Prepare to handle AS3 requests (i.e., load AS3 schemas). This runs asynchronously and the LX framework invokes onStartCompleted() afterward.

The iControl LX framework calls this method when instantiating the AS3 plugin. We indicate success or failure by invoking one of the callback functions supplied as arguments. (If we indicate failure then the framework will disable AS3.)

onStart(success, failure)¶
Arguments:
  • success (function) – callback to indicate successful startup
  • failure (function) – callback to indicate startup failure
Return undefined:

Function: onStartCompleted¶

Function: onGet¶

Return a copy of some declaration to the client. Caller appends [/target[/tenant[/expand[/age]]]] to WORKER_URI_PATH to indicate which declaration he wants to see. If target is “index” we return a list of targets for which we have declarations and how many for each.

(It’s okay for target to be IPv6, colons and all, see https://url.spec.whatwg.org/ )

Function: onPost¶

Function: onDelete¶

Attempt to remove one or more AS3 tenants from BIG-IP when caller uses DELETE method.

Caller’s authorization is not checked here but rather in doTheNeedful().
